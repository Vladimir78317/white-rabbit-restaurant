WHITE RABBIT v3.3 — ИСПРАВЛЕНИЕ БАЗЫ И ПРОЕКТА

ПОЧЕМУ БЫЛИ ОШИБКИ
В рабочей базе отсутствовали несколько процедур для интерфейса и таблица dbo.DishStopList.
Из-за этого приложение не могло загрузить меню, брони по дням и фильтры смен.

КАК ИСПРАВИТЬ ТЕКУЩУЮ БАЗУ БЕЗ УДАЛЕНИЯ ДАННЫХ
1. Закройте программу White Rabbit.
2. Откройте SQL Server Management Studio.
3. Подключитесь к тому же экземпляру SQL Server, который использует программа.
   По умолчанию приложение использует: Server=.; Database=WhiteRabbitRestaurant.
4. Откройте SQL_Repair_v3_3_AllMissingObjects.sql.
5. Нажмите Execute / F5 и дождитесь строки:
   «White Rabbit v3.3: все объекты для приложения восстановлены.»
6. Откройте SQL_Verify_v3_3_AllMissingObjects.sql и также нажмите Execute / F5.
7. В первой таблице проверки запомните значение «SQL Server». Оно должно совпадать с сервером,
   к которому подключается программа. После пересборки этой версии оно также записывается
   в файл white-rabbit-error.log при любой ошибке.
8. Запустите проект из Visual Studio заново.

ВАЖНО
- SQL_Repair_v3_3_AllMissingObjects.sql не удаляет существующие заказы, брони, пользователей и меню.
- WhiteRabbitRestaurant_Complete_Install_v3_3.sql предназначен только для чистой установки:
  он удаляет существующую базу перед созданием новой.
- Исходный проект обновлён: журнал ошибок содержит только имя SQL Server и имя базы,
  без пароля и других секретов подключения.
