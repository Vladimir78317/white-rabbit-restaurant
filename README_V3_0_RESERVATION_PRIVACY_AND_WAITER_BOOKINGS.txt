White Rabbit v3.0 — защита броней и визуальная карта бронирований официанта

Что изменено
1. Клиент видит в разделе «Бронирование» только собственные активные брони.
2. Клиент не может отменить чужую бронь: проверка выполняется на уровне SQL-процедуры.
3. Администратор сохраняет доступ ко всем броням: выбирает дату, нажимает столик-карточку и видит все брони этого столика за день.
4. Официант видит только свои назначенные столики карточками.
   После выбора даты и нажатия на карточку столика справа открывается вкладка «Брони столика» со всеми бронями этого столика за день.
5. В администрировании остаются фильтры смен по дате, официанту, статусу и типу смены.

Как установить обновление
1. Откройте SQL Server Management Studio.
2. Выберите базу WhiteRabbitRestaurant.
3. Если у вас установлена версия v2.8, выполните SQL_Update_v3_0_All_From_v2_8.sql.
   Если ранее уже выполнено обновление v2.9, достаточно SQL_Update_v3_0_Reservation_Privacy_And_Waiter_Bookings.sql.
4. В Visual Studio откройте решение WhiteRabbitRestaurant.sln.
5. Выполните Restore NuGet Packages, Clean Solution, Rebuild Solution и F5.

Легенда карточек
- зелёная — свободный столик;
- жёлтая — на выбранную дату есть брони;
- красная — столик занят активным заказом;
- синяя — столик назначен текущему официанту;
- бордовая — выбранный столик.
