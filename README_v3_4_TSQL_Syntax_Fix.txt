WHITE RABBIT v3.4 — ИСПРАВЛЕНИЕ СИНТАКСИСА SQL

Почему была ошибка:
В версии v3.3 в конце ремонтного скрипта использовалось выражение вида:
THROW ..., CONCAT(...), 1
SQL Server не допускает вызов CONCAT непосредственно в параметре THROW, поэтому выдавал ошибку 102 около CONCAT.

Что запускать:
1. Полностью закройте программу White Rabbit.
2. В SSMS подключитесь к тому экземпляру SQL Server, который используется программой.
3. Откройте SQL_Repair_v3_4_TSQL_Syntax_Fix.sql и выполните F5.
4. Откройте SQL_Verify_v3_4_TSQL_Syntax_Fix.sql и выполните F5.
5. После сообщения «Проверка v3.4 завершена успешно.» запустите программу заново.

Скрипт обновления не удаляет базу, пользователей, блюда, брони и заказы.
Он повторно создаёт недостающие таблицы и процедуры и выполняет совместимую финальную проверку.

Не запускайте файлы с суффиксом _OBSOLETE.sql: они сохранены только для истории и содержат ошибочный вариант финальной проверки v3.3.
