# Руководство пользователя: White Rabbit

## О программе

White Rabbit — настольное приложение для автоматизации работы ресторана. Программа предназначена для бронирования столиков, оформления заказов, передачи заказов на кухню, управления сменами официантов, оплаты заказов, учёта доступности блюд и формирования отчётов.

Приложение реализовано на языке C# с использованием технологии Windows Forms. Данные хранятся в Microsoft SQL Server. В системе предусмотрены роли клиента, официанта, сотрудника кухни и администратора.

Программа позволяет:
- регистрировать клиентов и выполнять вход в систему;
- бронировать столики с учётом вместимости;
- просматривать схему столиков;
- оформлять заказы из меню;
- передавать заказы на кухню и отслеживать статусы;
- открывать и закрывать смены официантов;
- автоматически распределять столики между работающими официантами;
- принимать оплату и автоматически освобождать столик;
- вести стоп-лист и пополнять остатки блюд;
- формировать отчёты по оплачиваемым заказам.

## 1. Интерфейс программы

После входа открывается рабочее окно, соответствующее роли пользователя.

### 1.1. Клиент

Клиент может зарегистрироваться, бронировать столики, выбирать столик на схеме, добавлять блюда в корзину и просматривать собственные заказы.

### 1.2. Официант

В рабочем месте официанта доступны кнопки:
- **Открыть смену / прийти на смену**;
- **Обновить столики**;
- **Создать заказ**;
- **Отправить на кухню**;
- **Принести заказ**;
- **Пробить счёт**;
- **Закрыть счёт**.

### 1.3. Кухня

Сотрудник кухни видит очередь заказов и изменяет их статусы: «Готовится» и «Готов к выдаче». Также доступен стоп-лист.

### 1.4. Администратор

Администратор управляет сотрудниками, сменами, распределением столиков, статусами заказов, складом и отчётами.

## 2. Начало работы

1. Установите Microsoft SQL Server и SQL Server Management Studio.
2. Выполните базовый SQL-скрипт проекта.
3. Откройте `WhiteRabbitRestaurant.sln` в Visual Studio.
4. Выполните `Restore NuGet Packages`, затем `Clean Solution` и `Rebuild Solution`.
5. Проверьте строку подключения в `Services/Database.cs`.
6. Запустите приложение клавишей `F5`.

Для отображения автоматически назначенных столиков официанта обязательно выполните файл `SQL_Update_v2_8_Waiter_Table_Assignment_Fix.sql`.

## 3. Работа клиента

### 3.1. Регистрация и вход

1. На стартовом экране выберите регистрацию.
2. Укажите фамилию, имя, телефон, логин и пароль.
3. После успешной регистрации войдите под созданными данными.

### 3.2. Бронирование столика

1. Откройте вкладку «Бронирование».
2. Введите данные клиента, дату, время и число гостей.
3. Выберите подходящий столик на схеме.
4. Нажмите «Забронировать».
5. Для отмены выберите бронь в списке и нажмите «Отменить бронь».

Столик нельзя выбрать, если он занят, уже забронирован на указанное время или не подходит по вместимости.

### 3.3. Оформление заказа

1. Откройте вкладку «Заказать».
2. Выберите доступный столик и укажите число гостей.
3. Добавляйте блюда кнопкой «Добавить».
4. Проверьте корзину и отправьте заказ на кухню.
5. Статус заказа доступен в списке собственных заказов.

## 4. Работа официанта

### 4.1. Открытие смены

Официант может открыть назначенную администратором смену либо самостоятельно прийти на смену. Для этого нажмите кнопку «Открыть смену / прийти на смену».

После открытия смены столики распределяются автоматически между всеми официантами с открытыми сменами. Самостоятельно открытая смена сразу отображается у администратора.

### 4.2. Работа со столиками

1. Нажмите «Обновить столики».
2. Выберите назначенный столик.
3. Укажите число гостей.
4. Нажмите «Создать заказ».

Если список столиков пуст, проверьте открытие смены, нажмите «Обновить столики» и попросите администратора нажать «Распределить столики».

### 4.3. Оформление, выдача и оплата

1. Добавьте блюда в текущий заказ.
2. Нажмите «Отправить на кухню».
3. После готовности выберите заказ и нажмите «Принести заказ».
4. После передачи заказа гостю нажмите «Пробить счёт».
5. Выберите способ оплаты.
6. Нажмите «Закрыть счёт».

После успешной оплаты формируется чек, заказ закрывается, а столик автоматически освобождается.

### 4.4. Закрытие смены

Смена не закрывается, пока есть незавершённые заказы. При раннем закрытии необходимо указать причину.

## 5. Работа сотрудника кухни

1. Откройте вкладку «Кухня».
2. Выберите заказ из очереди.
3. Изменяйте статусы: заказ принят → готовится → готов к выдаче.
4. После готовности официант увидит заказ в активных заказах.

Для стоп-листа выберите блюдо, укажите причину и добавьте блюдо в стоп-лист. Блюдо исчезнет из доступного меню клиента и официанта.

## 6. Работа администратора

### 6.1. Сотрудники

Администратор создаёт учётные записи официантов и сотрудников кухни. Удаление сотрудника выполняется как архивирование: история смен и заказов сохраняется.

### 6.2. Смены и столики

1. Откройте раздел «Смены и столики».
2. Выберите официанта и укажите время начала и окончания.
3. Нажмите «Запланировать смену».
4. Для ручного пересчёта назначений нажмите «Распределить столики».

Столики делятся поровну между официантами с открытыми сменами.

### 6.3. Статусы, склад и отчёты

- «Статусы заказов» — просмотр всех заказов и их текущего состояния.
- «Отчёт по продажам» — формируется только по оплаченным заказам за выбранный период.
- «Склад» — пополнение остатков блюд и просмотр истории операций.

## 7. Статусы заказа

| Статус | Значение |
|---|---|
| Новый | Заказ создан. |
| Передан на кухню | Заказ ожидает обработки кухней. |
| Готовится | Кухня готовит заказ. |
| Готов к выдаче | Официант может забрать блюда. |
| Передано гостю | Официант выдал заказ гостю. |
| Оплачен | Оплата принята. |
| Закрыт | Счёт закрыт, столик свободен. |

## 8. Возможные ошибки

| Проблема | Причина | Решение |
|---|---|---|
| Не удаётся войти в систему | Неверно введены логин или пароль, либо учётная запись архивирована | Проверить введённые данные. При необходимости обратиться к администратору. |
| У официанта нет столиков | Не обновлены назначения | Выполнить SQL-скрипт v2.8, открыть смену и нажать «Обновить столики». |
| Блюдо не добавляется | Блюдо в стоп-листе или нет остатка | Проверить стоп-лист и склад. |
| Не закрывается смена | Есть незавершённые заказы | Завершить и оплатить активные заказы. |
| Ошибка приложения | Ошибка данных или подключения | Открыть файл `white-rabbit-error.log` рядом с программой. |

## 9. Завершение работы

Перед выходом убедитесь, что все оплаченные заказы закрыты. Официанту необходимо закрыть смену, если по ней нет незавершённых заказов. Для выхода используйте кнопку закрытия приложения.

## 10. Заключение

White Rabbit объединяет работу клиента, официанта, кухни и администратора в одной системе. Соблюдение описанного порядка действий помогает корректно обрабатывать бронирования, заказы, смены и оплату.
